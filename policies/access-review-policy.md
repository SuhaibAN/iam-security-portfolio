# Access Review Policy

## Purpose

This policy defines how user access should be reviewed to ensure users only keep access required for their current role.

## Scope

This applies to employees, contractors, privileged users, and application owners.

## Review Frequency

| Access Type | Review Frequency |
|---|---|
| Standard user access | Every 6 months |
| Privileged access | Every 3 months |
| Contractor access | Every 3 months |
| High-risk applications | Every 3 months |

## Review Process

1. Generate a list of users and assigned access.
2. Send access list to managers or application owners.
3. Confirm whether access is still required.
4. Remove unnecessary access.
5. Document review decisions.
6. Escalate risky or unclear access.

## Review Decisions

| Decision | Meaning |
|---|---|
| Approve | User still needs access |
| Remove | User no longer needs access |
| Investigate | Access requires more review |

## Security Value

Access reviews reduce excessive permissions, privilege creep, and unauthorized access risk.
