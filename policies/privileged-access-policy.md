# Privileged Access Policy

## Purpose

This policy defines how privileged access should be requested, approved, monitored, and reviewed.

## Scope

This applies to administrator accounts, elevated permissions, and access to sensitive systems.

## Requirements

Privileged access should:

- Require manager or application owner approval
- Require MFA
- Be granted based on job need
- Be limited to the minimum access required
- Be reviewed every 3 months
- Be removed when no longer needed

## Approval Process

1. User submits privileged access request.
2. Manager reviews business need.
3. Application owner or security team reviews risk.
4. Access is approved or denied.
5. Approved access is assigned.
6. Access is included in future reviews.

## Security Value

Privileged access control reduces the risk of unauthorized system changes, data exposure, and account misuse.
